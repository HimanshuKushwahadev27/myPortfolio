Distributed Bidding

Bidding Simulator

A distributed bidding system built to practice real-time coordination, service boundaries, and state consistency under concurrent load.

  • Engineered a distributed auction engine with 3 concurrency strategies — Redis distributed locks, Optimistic Locking (JPA versioning), and Naive — benchmarking conflict rates from 0% to 40% under simulated high-load scenarios.
  • Built a concurrent bid simulator capable of firing N parallel bid requests, exposing real-time metrics (success rate, avg latency, conflict rate) to compare locking strategy trade-offs at scale.
  • Implemented Optimistic Locking with retry logic as the default strategy, achieving 4× higher throughput over Redis Lock while maintaining acceptable conflict rates (5–15%) for typical auction load.
